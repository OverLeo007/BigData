## 1 Задачи

Для достижения цели необходимо cпроектировать алгоритмы _MapReduce_, которые принимают на входе "очень большой файл целых чисел" (ОБФЦЧ) и порождают:  
- наибольшее число;
- среднее арифметическое всех чисел;
- набор тех же чисел, из которого исключены дубликаты;
- количество различных целых чисел во входном наборе.
## 3 Описание алгоритмов

### 3.1 Наибольшее число
#### Шаг 1. Разбиение входного файла

ОБФЦЧ хранится в распределённой файловой системе и заранее разбит на блоки, каждый блок обрабатывается отдельным mapper’ом.
#### Шаг 2. Работа mapper

Каждый mapper читает только свою часть файла и ищет максимум внутри неё.
На выход mapper выдаёт пару с одинаковым ключом.

#### Шаг 3. Shuffle and Sort

MapReduce группирует промежуточные пары по ключу.
Так как ключ у всех пар одинаковый, reducer получит один набор значений
#### Шаг 4. Работа reducer

Reducer проходит по полученным значениям и выбирает среди них максимум
#### Шаг 5. Результат

Итоговый выходной файл содержит одно число, то есть наибольшее число из всего ОБФЦЧ.
### 3.2 Cреднее арифметическое всех чисел

#### Шаг 1. Разбиение входного файла

ОБФЦЧ хранится в распределённой файловой системе и заранее разбит на блоки, каждый блок обрабатывается отдельным mapper’ом.
#### Шаг 2. Работа mapper

Mapper проходит по числам своего блока и считает две величины:

```
localSum   = сумма чисел в блоке
localCount = количество чисел в блоке
```
#### Шаг 3. Shuffle and Sort

MapReduce группирует пары по ключу.

Так как ключ один и тот же, reducer получает все частичные суммы и количества:

```
"avg" -> [(localSum1, localCount1), (localSum2, localCount2), ...]
```

#### Шаг 4. Работа reducer

Reducer складывает отдельно суммы и отдельно количества:

```
totalSum   = sum(localSum1, localSum2, ...)
totalCount = sum(localCount1, localCount2, ...)
```

После этого считает среднее:

```
average = totalSum / totalCount
```

Reducer записывает результат.
#### Шаг 5. Результат

Итоговый выходной файл содержит среднее арифметическое всех чисел из исходного ОБФЦЧ.
### 3.3 Набор тех же чисел, из которого исключены дубликаты
#### Шаг 1. Разбиение входного файла

ОБФЦЧ хранится в распределённой файловой системе и заранее разбит на блоки, каждый блок обрабатывается отдельным mapper’ом.
#### Шаг 2. Работа mapper

Mapper читает числа из своего блока. Для каждого числа он формирует пару, где само число используется как ключ:

```
(number, null)
```

Суть в том что при добавлении числа в map в качестве ключа может быть единственная (в рамках mapper-а) запись с таким ключом. Таким образом в конце шага в mapper-ах находятся локально уникальные наборы чисел.

#### Шаг 3. Shuffle and Sort

MapReduce группирует пары по ключу.

Если число содержится в результатах работы нескольких Mapper-ов, получим примерно такой результат:

```
number   -> [null, null, ...]
```

#### Шаг 4. Работа reducer

Reducer получает одно число как ключ и список значений для него. Затем записывает ключ.
Получившийся набор ключей не содержит дубликатов.
#### Шаг 5. Результат

Выходной файл содержит все различные числа из исходного ОБФЦЧ. Каждое число записано один раз.
### 3.4 Количество различных целых чисел во входном наборе

Логично сперва выполнить алгоритм 3.3 для получения уникальных чисел.

#### Шаг 1. Выполнение алгоритма исключения дубликатов

В результате выполнения получаем такой же ОБФЦЧ, но без повторяющихся значений. Он хранится в распределённой файловой системе и заранее разбит на блоки, каждый блок обрабатывается отдельным mapper’ом.

#### Шаг 2. Работа mapper

Для каждой записи mapper создает пару:

```
(distinct, 1)
```

#### Шаг 3. Shuffle and Sort

Все пары группируются по одному ключу:

```
"distinct" -> [1, 1, 1, 1, 1, 1, 1, 1, 1]
```
#### Шаг 4. Работа reducer

Reducer складывает все единицы.

#### Шаг 5. Результат

Выходной файл содержит кол-во различных чисел из исходного ОЧФЦЧ.
